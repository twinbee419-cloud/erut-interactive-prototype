## Context

ERUT 윈도우 프로그램의 현장 검사 시나리오에서 PC 다운·정전·앱 강제 종료로 인한 측정 데이터 손실 방지. NDT 데이터는 후처리 불가능한 1회성이므로 손실 시 재측정 부담이 크며 검사 일정 지연 발생.

---

## 사양

### 1. 자동 저장 주기 — **30초**

- 매 30초마다 메모리 버퍼 → 디스크 flush
- PRF 1000Hz × 30초 = 약 30,000 샘플 단위로 안전망 확보
- 디스크 IO 부담과 데이터 손실 위험 균형 (NDT 일반 권장)

### 2. 저장 매체 — **임시 .bin 파일**

CLAUDE.md C-SCAN 바이너리 포맷(.bin) 패턴 따름:

- **임시 파일 경로**: `ERUT_Data/Projects/{ProjectFolder}/ScanData/{ItemID}/Session_{NNN}/_autosave.bin`
- 정식 세션 종료(F7) 시: `_autosave.bin` → `cscan_chN_amp.bin` 등 정식 파일명으로 rename
- 임시 파일은 헤더에 `IsAutosave: true` 플래그 명시

### 3. 저장 시점 트리거

| 트리거 | 처리 |
|---|---|
| **자동 timer 30초** | 메모리 버퍼 → `_autosave.bin` append |
| **수동 저장 (Ctrl+S)** | 즉시 flush (timer reset) |
| **측정 중지 (F7)** | 세션 종료 + 정식 파일명으로 rename |
| **앱 정상 종료** | 마지막 flush + 임시 파일 정리 |
| **앱 비정상 종료** | `_autosave.bin` 그대로 보존 → 다음 시작 시 복구 |

### 4. 복구 절차 (앱 시작 시)

```
1. 앱 시작 → ERUT_Data 스캔
2. `_autosave.bin` 존재하는 세션 폴더 발견 시:
   a. 다이얼로그 표시: "이전 세션 #047 데이터 발견. 복구하시겠습니까?"
   b. 3-옵션:
      - "[11] 계속" → [11] RealtimeScan 진입 + 데이터 로드 + 일시정지 상태
      - "[7] 보고서" → [7] 데이터 관리로 진입 + 데이터 로드
      - "취소" → `_autosave.bin` 폐기 (사용자 명시 의도)
3. 복구 데이터 무결성 검증 (header CRC, 데이터 길이 일치 등)
4. 검증 실패 시: 경고 + 부분 복구 옵션 제공
```

### 5. UI 표시 (윈도우 프로그램)

- **[11] 툴바 우측**: "마지막 저장 N초 전" 텍스트 (자동/수동 구분 없이 가장 최근)
  - 0~59초: "마지막 저장 N초 전"
  - 60초 이상: "마지막 저장 N분 전"
- 기존 `F6 시작 · F7 중지` 단축키 hint는 삭제 (해당 위치 사용)

### 6. 디스크 공간 부족 처리

- 자동 저장 실패 시 (디스크 공간 < 100MB 또는 write 에러):
  - 경고 배너 표시 ([11] 상단 또는 글로벌)
  - 문구: "디스크 공간 부족 — 자동 저장 실패. 다른 위치로 저장 또는 데이터 정리 필요"
  - 검사자 액션: 임시 폴더 정리 / 외장 드라이브 변경 / 측정 중지 후 보고서

### 7. SQLite 메타데이터 동기화

- `TB_SCAN_SESSION` 테이블의 `Status` 컬럼: `Active` / `Paused` / `Completed` / `AutosaveOnly` (비정상 종료)
- `LastAutosaveTime` 컬럼 (UTC, ms) — 마지막 자동 저장 시점
- `DataSize` 컬럼 — 현재 데이터 크기 (Bytes)

## 구현 우선순위

| Phase | 작업 |
|---|---|
| **P0 (.NET 단계)** | 자동 저장 timer + .bin flush + SQLite 메타 sync |
| P0 | 앱 시작 시 미완료 세션 감지 + 복구 다이얼로그 트리거 |
| P0 | 디스크 공간 모니터링 + 경고 배너 |
| P1 | 사용자 환경 설정 (자동 저장 주기 변경 — 10초~5분) |
| P1 | 부분 복구 (헤더 손상 시 데이터 부분 추출) |
| P2 | 외장 드라이브 자동 백업 (선택) |

---

## 윈도우 vs 웹 컨텍스트

윈도우 프로그램의 현장 단기 검사 컨텍스트에서 매우 의미 있는 기능. **웹 서비스(상시 모니터링)와는 별개** — 웹은 자체 백엔드 DB 저장 (윈도우 자동 저장과 무관).

데이터 동기화: 윈도우 → 웹 업로드 사양은 별도 (현재 미정).
