## 문서 작성 원칙

1. **개발 구현 가능 수준의 상세도**: 알고리즘·데이터 구조·예시·테스트 케이스 포함
2. **NDT 도메인 지식 명시**: 기획팀 외 인원도 이해 가능한 도메인 설명 포함
3. **버전 관리**: 파일명에 `_vX.Y` 명시. 변경 시 새 버전으로 추가
4. **변경 이력**: 문서 마지막에 변경 이력 섹션 유지
5. **미해결 항목 명시**: 사용자/개발팀이 확정해야 할 사항은 마지막 섹션에 정리
