## 1. 원칙

DAQ 연결(연결 테스트 = LAN TCP 핸드셰이크) 시, **장비가 스스로 아는 값은 자동 획득**한다. 사용자가 직접 입력하는 값은 **네트워크 접속 정보 + 사용자 라벨**뿐이다.

- **사용자 입력**: 별칭(alias) · IP · Port · Timeout · 유형(기대 OEM 모델)
- **자동 획득(연결 시 회신)**: SN · 채널 수 · 펌웨어 등 아래 표

이로써 수동 입력 오타·불일치를 원천 제거한다. (유형만 사용자가 기대값을 고르고, 실제 모델과 다르면 경고.)

---

## 2. 자동 획득 항목 (하드웨어 진단 → 시스템 정보)

| 항목 | 자동 획득 | 출처 | 비고 |
|---|---|---|---|
| 시리얼 (SN) | ✅ | 핸드셰이크 회신 | [4-1] SN 필드도 이 값으로 자동 채움 |
| 펌웨어 | ✅ | 핸드셰이크 회신 | [4-1] 펌웨어 필드 자동 채움 |
| 하드웨어 리비전 | ✅ | 펌웨어 보고 | |
| 샘플링 속도 | ✅ | 보드 스펙(펌웨어 보고) | |
| 최대 채널 수 | ✅ | 핸드셰이크 회신 | [4-1] 채널 수 필드 자동 채움 |
| IP 주소 | ✅ | 접속에 사용한 값 | 사용자 입력 IP:Port 그대로 |
| MAC 주소 | ✅ | 네트워크 레이어 조회(ARP 등) | |
| Config 파일 | ✅ | 보드 현재 Config 조회/다운로드 | |
| **제조일** | **△ 조건부** | 펌웨어 제조 메타데이터 | **펌웨어가 제공하면 표시, 없으면 `—`** |

> **제조일 fallback**: 펌웨어가 제조일 메타를 노출하지 않는 DAQ는 `—`로 표시. UI에 `value || "—"` 처리 반영됨. (그 외 항목도 미제공 시 동일하게 `—`.)

## 3. UI 반영

- 하드웨어 진단 '시스템 정보' 헤더에 안내: **"DAQ 연결 시 자동 조회 · 미제공 항목은 —"**.
- [4-1] DAQ 폼: SN·채널 수·펌웨어는 읽기전용 자동 채움(`is-disabled`), '연결 후 자동 감지' placeholder.
- 연결 진단(연결 테스트 결과)에는 SN·펌웨어·채널 수를 별도 표시하지 않음 — 자동 채움된 폼 필드가 그 역할(중복 제거).

---

## 4. 개발팀 확인 필요

- **제조일 메타 노출 여부**: 대상 OEM 보드(OEM-PA2·PAmini·MC2·MCu·PAmax·MCuF·PAmicro) 펌웨어가 제조일을 회신하는지 확인. 미회신 모델은 `—`.
- **핸드셰이크 프로토콜**: SN·펌웨어·채널 수·HW rev·샘플링 속도·Config를 어떤 명령/응답으로 얻는지 정의 필요.
