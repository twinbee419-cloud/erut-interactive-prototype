## Context

ERUT는 자석 부착 로봇 + UT 보드 + 64ch 탐촉자 기반의 **고정형 상시 초음파 검사 시스템** (수조탱크·파이프·압력 용기 등). 현재 v8.10까지 18 슬라이드 11화면 + 5 모달이 설계되어 있으며, NDT 표준(KS B 0817 / ASNT / ASME B31.3 / API 510 등) 기반 워크플로우(장비 구성 → 교정 → 검사 → 판정 → 보고서)는 큰 줄기에서 모두 커버됨.

본 의견서는 **NDT 도메인 전문가 관점**에서 현재 플로우의 누락·혼란·중복을 발굴하고, 각 항목에 대해 **근거(NDT 표준/관행)와 효과**를 명시한다. 우선순위는 (1) 부족한 기능, (2) UX 혼란, (3) 불필요한 전환 순.

---

## 1순위 — 부족한 기능 (Critical Missing)

NDT 표준상 필수이거나 현장 안전·신뢰성에 직결되는 누락 기능.

### 1.1 DAC / TCG 곡선 관리 화면 ★★★

| 항목 | 내용 |
|---|---|
| **현황** | [2] 셀 메타 라인에 "DAC 대비" 값만 표시. 곡선 작성/편집 UI 없음 |
| **부족** | 거리(=ToF)별 신호 감쇄 보정 곡선 — 같은 결함이라도 거리에 따라 진폭이 달라지므로 보정 필수 |
| **근거 (NDT)** | ASME V Article 4 T-432 (DAC 작성 절차), KS B 0817 §7.4. 참조 반사체(SDH·FBH) 3점 이상으로 곡선 산출. 표준 인증의 기본 절차 |
| **권장** | [3] Gate 설정 페이지에 "DAC 곡선" 서브 탭 추가. 표준 템플릿(ASME T-481.1 등) 로드 + 측정점 입력 + 그래프 시각화 + ±n dB 임계선 |
| **효과** | (a) 거리에 따른 결함 누락 방지, (b) 표준 인증 가능 (KS·ASME 호환), (c) 검사자 주관 보정 제거 → 측정 신뢰성 ↑ |

### 1.2 결함 판정 기준 (Acceptance Criteria) 설정 화면 ★★★

| 항목 | 내용 |
|---|---|
| **현황** | Amp threshold는 Gate width/threshold에 포함되지만 분산. 두께 감육 기준·결함 등급(A/B/C)·합격/불합격 자동 판정 규칙 정의 화면 없음 |
| **부족** | NDT의 최종 출력은 "합격/불합격" 판정인데 그 기준이 화면화되어 있지 않아 검사자 주관 개입 가능성 |
| **근거 (NDT)** | ASME B31.3 §344, API 510, KS B 0817 §9 — 각 표준별 판정 기준 상이 (Amp ≥ 50% DAC = recordable, ≥ 100% DAC = reject 등). 두께는 공칭 두께의 80% 이하 시 reject 일반적 |
| **권장** | [6] 검사 대상 관리에 "판정 기준" 섹션 추가 — 검사체별로 (a) Amp threshold (% DAC), (b) 두께 최소값 (mm or % nominal), (c) 결함 등급 매핑 (Critical/Major/Minor). [11] 실시간에서 자동 판정 + 색상 표시 |
| **효과** | (a) 합격/불합격 자동 판정 → 검사자 부담 ↓, (b) 보고서 표준 부합 명확화, (c) 동일 대상에 대해 검사자 간 판정 일관성 확보 |

### 1.3 결함 발견 시 실시간 알림 (Critical Alert) ★★★

| 항목 | 내용 |
|---|---|
| **현황** | P1 계획 "결함 발견 알림" — 상세 사양 없음. [11] 실시간에 Critical 배너 mockup만 존재 |
| **부족** | 64ch 동시 모니터링 시 검사자 시각이 모든 채널을 동시에 못 봄. 임계값 초과 채널이 발생해도 놓칠 수 있음 |
| **근거 (NDT)** | 고정형 상시 모니터링은 "anomaly가 발생하면 즉시 알린다"가 핵심 가치. 항공·플랜트 SHM(Structural Health Monitoring) 시스템의 필수 기능 |
| **권장** | [11] 우상단 Critical 배너 + 알림음 + 시스템 트레이 알림. 임계값은 1.2의 판정 기준에서 자동 가져옴. 알림 발생 → 해당 채널로 자동 포커스 (A-scan 전환) |
| **효과** | (a) 결함 검출 누락 방지, (b) 즉시 대응 가능, (c) 무인 야간 운영 시 SMS/메일 연계까지 확장 가능 |

| **권장** | [0-0] 프로젝트 생성 모달에 검사자 ASNT Level 입력 필수. [18] 보고서 서명, [4-3-1] 교정 권한, 판정 기준 수정 권한 등을 Level별 분기 |
| **효과** | (a) 표준 부합, (b) 검사자 책임 명확화 |

### E. 검사 보고서 다국어 / 표준 다중 지원

| 항목 | 내용 |
|---|---|
| **현황** | [18] 보고서 자동 생성 — 한국어 KS 표준 기반 |
| **권장** | 영문 보고서 옵션 (ASME V / ASNT 양식), 일문 (JIS Z 2305). [0-0] 프로젝트 생성 시 보고서 언어 선택 |
| **효과** | (a) 글로벌 현장 적용 가능, (b) 해외 고객 대응 |
