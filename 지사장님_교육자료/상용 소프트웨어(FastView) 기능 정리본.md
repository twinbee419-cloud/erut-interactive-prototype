# 상용 소프트웨어(FastView) 기능 정리본

## ★ 핵심 요구사항 · 필수 구현 (요약)

### 【1부 — 프로젝트·검사 운용 요구】

**최우선 핵심 (원문이 ’핵심’으로 명시)**
- 분할 스캔 파일(약 30개)을 **단일 파일로 병합** (1부 §6·§7)
- 병합 후 **일괄 후보정 동시 적용** — 최대 30개 파일 동시 증폭 (1부 §7)
- **검사 시작 신호 입력** (Sanction Position 대신 두산에너빌리티에 필요한 트리거) (1부 §5)

**검사 구성·파라미터**
- 검사 방식 = **갠트리**(제품 고정·프로브 이동) (1부 §4)
- 검사 대상 = **쉘(실린더형) → 파이프 유형**(반지름/지름) (1부 §5)
- **로봇 팔 + 리프트 자동 스캔**(0~4,000mm 구간 통합)·구역 분할 (1부 §6)
- 원주 방향 **접촉 센서 2개**·**지름 기반 오버랩 자동 계산**(분할 개수 자동) (1부 §6)
- **X축 500mm 단위 분할 스캔** 후 병합 (1부 §5)
- **곡률 각도 입력 + SOI(전개도)** — 원형 아닌 다각형 실린더 지원 (1부 §5)
- **End of Bench**(이동 한계) · **엔코더 기반 선형/최대 속도**(데이터 누락 방지) (1부 §5)
- **단위(mm) 변환 오류 수정** · 데이터 표현 X/Y 좌표·파이프라인 자동 변환 (1부 §5)

**교정·탐촉자·데이터**
- **교정 주기 4시간 + 알람**(원자력 법규 · 검사자 직접 수행) (1부 §9)
- 탐촉자 **8개 개별 홀더**(두산에너빌리티 문서 기준 사양) (1부 §8)
- 수집 데이터 **미니 컴퓨터로 전송** (1부 §8)

**검사 대상 규모**
- SMR 실린더: 높이 최대 6m(통상 3~4m) · 지름 2~3m · 검사 면적 4m×12m (1부 §6)

**제외 (철강 자동화 전용)**
- Sanction Position · ‘섹션 위치’ 정보 · 외부 센서(PA 장비 외) 관련 항목 (1부 §5)

**미결 (하드웨어 미완성)**
- 마킹 시스템(시작·중단·재개 지점) · 물 공급 모터 제어 (1부 §10)

### 【2부 — 상용 SW 기능·개발 요건】

**두산에너빌리티 요구 · 법규 핵심**
- **글로벌 dB 사후 조정** — 저장된 로우 데이터 기반으로 dB 변경 시 **전체 C-scan 실시간 반영**(포토샵 밝기 방식). 법규상 ’가장 좋은 부분 80% 기준’을 검사 전엔 알 수 없어 **검사 후 조정 필수**. 수동은 수백 격자 개별 조정으로 1건 1주일 → **자동화가 프로젝트 핵심 가치**. 항공우주 확장 가능 (2부 §5)
- **자동 교정(Calibration)** — 현재 수동 → 자동화. 첫/둘째 에코 높이 80% 일괄 조정. 두산에너빌리티 측은 **시료판 3점 이상 자동 교정** 요구 (2부 §4)
- **TCG 자동 교정** — 깊이별 감쇠 보정 슬로프 자동 생성. **구매 사양서 필수 명시** (2부 §4)
- **엔코더 캘리브레이션 자동화** — 실제 이동 거리 입력 → 로우 값/펄스 자동 계산 → 카운트 자동 업데이트. **고객 사양서 요구** (2부 §3)
- **한 바퀴 회전 전체 데이터를 하나의 파일로 펼쳐 시각화** + 감도 저하 구간 등 특이사항 명확 표시 (2부 §5)
- **오스테나이트 스테인리스강 신호 감쇄 구간**을 정확히 탐지·시각 표현 (2부 §4)
- **검사 후 합격/불합격 자동 판정** + 바코드로 검사 일련번호(SN) 자동 매칭 (2부 §5)
- **평가: 결함 면적·위치·전체 대비 비율 측정**(수동 커서 지정) (2부 §5)

**개발 방향·구조**
- **프로브 개수 제한 두지 않도록** 개발(현재 최대 8개) (2부 §2)
- **고정형·이동형 통합 지원**(하나의 SW) (2부 §5)
- **범용성·확장성** — 두산에너빌리티는 보유 전 장비·SW를 이 시스템으로 통합 계획 → 초기부터 확장 고려 (2부 §5)
- **필수 뷰 6종**: A-scan · B-scan · C-scan · D-scan(=S-scan) · FFT · Strip Chart (2부 §5)
- **자유로운 화면(뷰) 구성** — 검사창 최대 5개·탭 추가, 뷰별 신호·프로브 할당, 교정/검사/평가 탭 (2부 §5·§4)
- **타임 슬롯 사용자 직접 지정**(자유도↑) (2부 §4)
- **자동 마킹**(실시간 신호 변화 → 외부 출력 → 마커 연동) (2부 §1·§4)
- **리포트** — 법적 증거력(A-scan·설정값 포함), 커스텀/엑셀, 상용화 수준, 직관·미려 (2부 §5)
- **계정 권한 분리** — 초음파 전문가(세팅 변경) / 현장 작업자(변경·삭제 불가) (2부 §5)
- **AI 대화형 매뉴얼**(한글, 인터넷 필요) (2부 §5)

**참고**
- 2026년까지 임시 라이선스 발급 및 SW 사용 지원 예정 (2부 §4)

## 2. 장비 접속 및 신호 표현 방식 (A-scan / B-scan / C-scan)

**장비 접속·기본 설정**
- **PA 미니, MC, Micro F2, Micro 플랫** 등의 장비를 선택하고 **IP 주소를 통해 접속**하여 기본 설정을 수행.
- **커넥트(Connect) 버튼**을 누르면 신호가 활성화됨.

**A-scan**
- 가로축이 **시간(0~30 마이크로초, μs)**, **신호 진폭을 파형**으로 표시.
- A-scan 화면을 통해 0~30μs 사이의 시간대별 신호 변화를 확인.

**B-scan**
- A-scan의 **진폭 값을 색상으로 변환한 2D 표현**(동일 데이터를 색상으로 매핑).
- 색상 매핑: **0 = 빨간색, 최댓값 = 파란색, 중간값 = 그라데이션**. (진폭 0~100 범위를 색상 스펙트럼에 매핑)

**BMP 활용 및 A-scan 역추출**
- 일부 고객사는 **B-scan·C-scan 데이터를 BMP(256 색상값 / 256단계) 파일 형식으로 저장·관리**.
- **BMP 픽셀의 높이값(색상 값)과 A-scan 진폭을 연동**하면, **이미지 파일로부터 역으로 A-scan 데이터를 추출**할 수 있음(고객사 활용 중).

**[Q&A]**
- Q: 위로 튀어오른 신호가 색상으로만 표현된 것인가?
- A: 맞음. 0~100 범위의 진폭 값을 색상 스펙트럼에 매핑한 것.

## 3. 디스플레이(뷰) 구성

## 4. 프로젝트·벤치(Bench) 설정 구조

**설정 순서**: 프로젝트 파일 생성 → 벤치(Bench) 정의 → 검사 대상 및 스캔 파라미터 입력.

- 이 설정 체계는 **핵심 고객인 철강 회사의 자동화 검사 시스템을 기준으로 설계**되어, **두산 에너빌리티 프로젝트에는 불필요한 항목이 다수 포함**됨.

**프로젝트**
- **Create / Load / Save** 기능으로 설정 파일 관리(검사 프로젝트 생성·관리 메뉴).
- 탐촉자 초기 세팅을 위해 **프로젝트 단위로 설정 파일을 생성**.

**벤치(Bench) 정의**
- **벤치 = 철강 산업 현장에서 검사 장비(검사 기계)를 지칭**하는 용어. 벤치 설정을 통해 **시스템 구성과 검사 대상의 형태를 정의**.
- 정의 항목: **벤치(검사 기계) 이름**, **검사 대상 제품명(예: SML 101, 규격에 따라 지정)**, **검사 방식 유형** 지정.
- 검사 제품을 지정하는 과정에서 **C-scan 기능 구현의 필요성**이 언급됨.

**검사 방식 (프로브·제품의 상대 움직임 유형)**
- 프로브 고정·제품 이동 / **제품 고정·프로브 이동(갠트리)** / 제품 회전·프로브 이동 등 다양하게(고정형·이동형·회전형) 설정.
- **두산 적용 방식 = 갠트리(Gantry): 제품 고정, 프로브 이동.**
- 갠트리 방식에서는 **프로브의 좌표 변화를 기준**으로 설정.

## 5. 검사 대상 형태 및 스캔 파라미터

**검사 대상 형태 (선택에 따라 프로브 좌표 정의 방식·입력 파라미터가 달라짐)**
- 파이프, 바(Bar), 용접 파이프, 평판(Plate) 등.
- **파이프·원형: 반지름 / 지름 입력** · **평판(Plate): X/Y축 길이 입력**.
- 두산 적용 대상 = **쉘(Shell, 실린더형)** → **파이프 유형으로 설정**.

**스캔 범위·축 파라미터**
- **End of Bench**: 프로브가 이동할 수 있는 **검사 한계 거리(범위)** 지정. 프로브 이동 한계 거리를 위해 **스캔 시작점·종료점 설정**.
- **Scanning Axis**: 두산의 경우 **X축을 500mm 단위로 짧게 분할 스캔**한 뒤 파일을 병합(개별 파일을 추후 하나로 통합).
- **Linear Speed / Max Speed**: **엔코더 신호 기반** 속도. **속도 초과 시 데이터 누락(구멍)이 발생**하므로, **실제 최고 이동 속도 입력이 필수**(데이터 누락 방지).

**철강 자동화 전용(두산 제외) 항목**
- **Sanction Position** 및 센서 관련 항목 = **철강 자동화 라인 전용 기능**으로 **두산 프로젝트에는 불필요**.
- 두산 프로젝트에서는 **검사 시작 신호만 필요**하므로 **Sanction Position 기능 제외**.
- 논의된 **센서 관련 설정 항목들은 PA 장비 외의 외부 센서 운용을 위한 사양**.
- 자동화 시스템용 프로그램 특성상 현재 **불필요한 ‘섹션 위치’ 정보 등은 개발 범위에서 제외**.

**데이터 표현·좌표·단위**
- 데이터 표현 시 **X/Y 좌표 설정** 및 **파이프라인 변경에 따른 자동 변환 기능** 구현 필요.
- 두산 프로젝트를 위해 **곡률 각도 입력** 및 **SOI(Surface of Interest, 전개도) 설정 기능** 필요.
- 기존 프로그램의 **단위 변환 오류를 수정하여 밀리미터(mm) 단위가 정상 적용**되도록 개선 필요.

**[Q&A]**
- Q: 두산에서 검사 방향이 위아래로 왕복하는 방식인가?
- A: **로봇 팔이 상승 후 원점 복귀, 리프트 상승을 반복하는 방식으로 0~4,000mm 구간을 연속 스캔**.
- Q: 곡률 각도 입력 및 전개도(SOI) 표현이 필요한가?
- A: **원형이 아닌 ‘대나무 물통’ 형태의 다각형 실린더도 지원 가능**하며, 해당 형태로 표현되어야 함.

**[강사 강조]**
- Sanction Position 등 철강 자동화 전용 항목은 두산 프로젝트에서 제외 가능하며, **두산에서 필요한 것은 ’검사 시작 신호 입력’이 핵심**.

## 6. 두산 에너빌리티 SMR 실린더 스캔 운용 시나리오

**검사 대상**
- **SMR(소형모듈원자로) 제작의 기초 재료인 대형 실린더**(쉘).
- 규모: **높이 최대 6m(통상 3~4m), 지름 2~3m** → 한 번에 검사하기 어려움.
- 한 기(基) 전체 스캔 대상 검사 면적: **4m × 12m 범위**(기기당 4m~12m 범위).

**분할 스캔 절차 (로봇 팔 + 리프트)**
- 검사 효율을 위해 **실린더 전체를 임의의 지점을 기준으로 여러 구역으로 분할**하여 검사.
- **로봇 팔이 검사 구역을 투영하며 상승 스캔 → 상단 도달 시 원점 복귀 → 리프트 상승**을 반복.
- **리프트와 로봇 팔의 동작을 연동**하여 **0~4,000(mm) 구간을 하나의 데이터로 통합**.
- 검사 완료 시 **리프트·로봇 팔이 자동으로 원점 복귀**하여 다음 검사를 준비.

**원주 방향·오버랩**
- 원주 방향: **접촉 센서 2개로 동일 간격 이동**.
- **탐촉자를 일정 간격으로 이동시키며 전체 영역을 스캔**하고, **입력된 지름에 따라 오버랩(중첩) 구간을 자동으로 계산**.
- **오버랩 적용 시 분할 개수 자동 계산 필요**.
- 검사 경로: **원점에서 시작해 순차적으로 이동하며 전체 영역을 중첩 방식으로 스캔**.

**파일 생성**
- 현재 장비는 **각 스캔 경로마다 독립적인 파일을 생성** → 스캔 완료 시 **약 30개의 독립 파일** 생성.

**스캔 속도·전력**
- 스캔 속도에 따른 **전력 소모와 배터리 운용 방안**을 검토. (※ “4m×12m”는 **검사 면적**이며 초당 이동 속도가 아님 — 위 ‘검사 대상’ 참조)

**[Q&A]**
- Q: 초당 스캔 속도 기준으로 4m~12m 면적 처리 시간 예측이 가능한가?
- A: **스캔 밀도와 속도에 따라 계산 가능**하며, 이를 기반으로 **교정 알람 타이밍 예측**도 고려 가능.

## 7. 데이터 통합·후보정 요구사항 (두산 핵심 스펙)

# 2부. 상용 SW 기능·개발 요건 심화

> **목적:** 기존 상용 SW의 전체 기능 흐름(벤치 → 프로브 홀더 → Input/Output → A-scan 교정 → C-scan 평가 → 리포트)을 이해하고, 신규 ERUT 스캔형은 이 수준 **이상**으로 개발.
> 

## 1. 소프트웨어 전체 구조 및 벤치(Bench) 설정

기존 상용 소프트웨어는 크게 **벤치(Bench) · 프로브 홀더(Probe Holder) · Input/Output · Marker** 네 항목으로 구성.

- **벤치(Bench)** = **장비가 움직이는 최대 범위**와 검사 대상의 형상을 정의하는 **‘빈 검사 기계’ 개념**. 검사 재료의 유무와 관계없이 장비가 이동할 수 있는 최대 범위를 의미하며, 실제 검사 대상이 벤치 내에 들어오면 그 안에서 세부 설정(전체 크기·대표 음속값 등)이 이루어짐.
    - 교정 전 **표준 음속값**을 지정. 퀵 세팅·보안 관련 설정은 현재 단계 제외.
- **복수 NDT 장비 동시 제어** 가능 — 하나의 PC로 다수 PA 보드를 동시 제어하여 넓은 면적을 효율 처리. 실제 중국 납품 사례에서 **PA 보드 109개를 단일 PC로 제어**.
- **NDT 디바이스 설정**: 장비 종류(Mini · Micro · Max · Pilot · Pilot Plus 등) 선택, **IP 주소 및 시리얼 넘버** 등록. 장비 목록에서 파일럿 등 연결 장비의 SN·IP 확인·추가.
- **마스터-슬레이브 동기화**: 복수 장비 운용 시 **마스터 장비 하나가 트리거 신호를 수신 → 나머지 장비에 배포**. **동시 발사** 또는 **순차 대기** 방식 선택 가능. (여러 장비를 하나의 장비처럼 동기화)
- **Input/Output**:
    - 신호 입력은 기본적으로 **엔코더 신호** 사용, 필요 시 **별도 I/O 카드 추가**(로봇·HGV 통신용 — 장비 자체 I/O 외).
    - **Output**: 게이트 **Threshold 초과 신호 발생 시** 외부 장치(**PLC** 등)/통합 PLC·PC로 **합격/불합격 신호 송출·판정**.
- **Marker**: Output과 연동하여 **결함 위치에 페인트 마킹**(분사) 등 물리적 표시 수행.

> 인용: “여러 개의 장비를 하나의 장비처럼 작동하게 만들기 위해서 사용하는 겁니다.”
> 

**[강사 강조]** 벤치는 **검사 기계 자체의 최대 범위**이며, **검사 대상의 크기와 혼동하지 말 것**.

---

## 2. 프로브 홀더(Probe Holder) 설정 및 오프셋 계산

프로브 홀더 = **복수의 탐촉자(프로브)가 장착된 구조체**. 홀더 자체의 좌표(기준점)와 각 프로브의 개별 오프셋을 정확히 입력해야 **C-scan 원점이 올바르게 계산**됨.

- **원리 예시**: 5개 프로브가 장착된 홀더에서 **중앙 프로브는 수직**, **주변 프로브는 45도 각도**로 초음파 발사 → 각 프로브가 실제로 검사하는 지점은 프로브의 물리적 위치와 다름. 소프트웨어가 이 **각도와 오프셋을 자동 계산**하여 C-scan을 정확히 그림.
    - 프로브의 형태·규격에 따라 **C-scan의 시작점이 달라지므로 반드시 고려**.
    - 5개 탐촉자가 각각 독립적으로 움직이는 경우, 각 프로브 위치에 따라 **개별 C-scan 생성** 또는 **데이터 오버랩 처리**.
    - 기존 **두산중공업 테스트 데이터**를 바탕으로 홀더 좌표·프로브 위치 설정.
- **홀더 좌표**: 기준점 **(0, 0, 0)** 으로 설정(기준점이므로 기본값 0). **회전(Rotation) 각도** 입력 가능. 센터 포지션·홀더 로테이션 각도로 전체 프로브 정렬 조정.
- **프로브별 설정 항목**: 이름, 타입, **시리얼 넘버**, **중심 주파수(Central Frequency)**, 채널 번호, **송수신 모드(펄스 에코 / 투과법)**, **X·Y 오프셋**, **Axial Angle(축 각도)**.
    - **중심 주파수 입력 시** 관련 파라미터가 **자동 연동 변경**.
    - **송수신 모드**: 단일 프로브 = **펄스 에코**, 두 프로브 = **송수신 분리(투과법)**. 상단에서 **송신(Emission)·수신(Reception)**을 개별 또는 통합 제어하는 모드 선택.
    - **X축 오프셋** 입력 → C-scan 데이터에 자동 반영.
    - **곡면 검사**를 위해 프로브별 **축 각도(axial angle)** 설정으로 실제 형상에 맞게 보정. 중앙 제외 주변 프로브에 개별 각도 부여 → 정밀 스캔.
    - **라벨**: 일련번호·프로브 종류 입력. **종파/횡파 등 파형 선택 시 라벨 이름 자동 생성**(수정 가능).
    - **지오메트리 설정**으로 주파수·그래픽 표현 방식 조정(시각 정보 제어).
    - (예) 두 번째 프로브는 이더넷(Ethernet) 방식 설정, X축 위치 값 조정으로 포지셔닝. 파형 유형 ‘종파 2번’ 자동 설정 등.
- **최대 8개 프로브 추가 가능** (세부 설정 메뉴). **개수 제한 완화(제한 없음) 요청**.
- **Connectivity**: 복수 장비 운용 시 **드롭다운에서 해당 장비 및 채널 선택**.
- (프로그램은 20년간 축적된 사용자 피드백을 바탕으로 개발되어 상세 설정 제공.)

**[학생 질의]** 프로브 오프셋 값이 C-scan에 어떻게 반영되는가?
**[강사 답변]** 각 프로브의 위치·각도에 따라 C-scan 원점이 달라지므로, **오버랩 표시를 원할 경우 전체 C-scan의 오프셋이 프로브별로 자동 보정**된다.

**[강사 강조]** 프로브 **개수 제한을 두지 않도록** 개발 시 반영할 것.

## 3. Input 설정 — 엔코더 및 로봇 좌표 연동

ERUT 스캔형 시스템에서는 **물리적 엔코더 바퀴 대신 로봇의 6축 좌표를 Modbus 통신으로 실시간 수신** → 별도의 변환 프로그램이 **X·Y·Z 3축으로 변환** → 로봇이 **1mm 이동할 때마다 엔코더 펄스와 동일한 신호를 생성**하여 NDT 장비로 입력. 이 과정에서 **IO 박스가 중간에 위치**. (바퀴형 엔코더 장치 불필요)

- **연결 순서**: 컴퓨터(로봇 컴퓨터) → 변환 프로그램 → IO 박스 → NDT 장비.
- **엔코더 타입**: **Quadrature(쿼드러처)**. **X축·Y축 각각 엔코더 Input 추가**.
- **핀 맵 기준**: 장비 IO 1~6번 채널 중 해당 핀 번호 입력. SDK로 입력 채널 지정, 하드웨어 핀맵 참조해 엔코더 출력단과 장비 채널 일치. **인풋 채널 번호는 사용자가 변경 가능하도록 구현 필요**.
    - (현재) A상 신호 = 2번 IO, B상 신호 = 3번 IO에 연결.
- **방향 전환**: 로봇 이동 방향과 값(좌표) 증가 방향 불일치 시 **소프트웨어 방향 전환 기능으로 보정**. (SDK 엔코더 주소 설정 항목)
- **다축**: 기본 스캔 축 X축 외에 추가 엔코더 입력으로 다축 스캔 구성.
- **Y축**: **로봇 이동 + 리프트 이동 합산 값**이 Y축 엔코더 입력으로 사용(좌우 이동 + 리프트 상승 합산 → 두 엔코더 입력 채널 구성).
- **엔코더 캘리브레이션**(개발 요청):
    - 엔코더는 **사용 중 마모로 측정값이 변할 수 있어 주기적 캘리브레이션 필요**.
    - 방식: **실제 이동 거리 입력 → 로우 값 자동 계산 → 카운트 값(펄스/mm) 자동 업데이트**. 엔코더 리셋 후 특정 거리 이동 → 로우 밸류 계산 → 카운트 자동 업데이트.
    - 캘리브레이션 값 예: **1mm 이동 시 펄스 수 13.85 스텝/mm** 기준. 엔코더 사양 **1.375** 입력으로 실제 이동과 일치 보정. **바퀴 크기·물리 환경에 따라 변동되는 고정 아닌 값**.
    - **엔코더 카운트 수동 입력 대신 실제 이동 거리 기반 펄스 자동 계산** 교정 기능 개발 필요 — **고객 사양서 명시 요구**.
- **로봇 통신 속도**: **UR(유니버설) 로봇 vs 3S(쓰리에스) 로봇**의 통신 리프레시 속도 차이 존재 → 3S 측 지원으로 동기화(전송 속도·주기) 조정 예정. UR과 인터페이스가 유사한 **저가형 로봇** 도입 과정의 통신 속도 차이는 해당 업체 기술 지원으로 인터페이스 최적화 진행.
- (현황) 현재는 **프로브 2개 + 엔코더 1개**로 가로 방향 이동만 가능. 엔코더 초기값 미설정으로 실제 이동 거리와 화면 표시 거리 간 오차 발생 → 설정 조정 메뉴 확인 필요. 로봇 X·Y 반복 이동으로 C-scan 신호 순차 획득.
- **두산에너빌리티 측 요구에 따라 아웃풋(Output) 설정 변경 가능성** 염두.

**[학생 질의]** 로봇에서 나온 선이 장비에 직접 연결되는가?
**[강사 답변]** **컴퓨터 → 변환 프로그램 → IO 박스 → NDT 장비** 순으로 연결된다.

## 4. A-scan 세부 설정 및 교정(Calibration)

A-scan = 초음파 신호의 기본 표시 단위. **게인(아날로그/디지털) · 샘플링 주파수 · PRF · 딜레이 · 랭스 · TCG · 필터 · 평균화(Average) · 펄스 전압** 등 다수 파라미터 설정 필요. (장비-파일럿 연결 후 A-scan 생성은 SDK 필수 설정 단계. OMPA 툴 데모의 UT 세팅 버튼으로 상세 설정 진입.)

**게인(Gain)**
- **아날로그 게인** = **하드웨어 보드 단에서 물리적으로 수신 신호를 증폭**.
- **디지털 게인** = **소프트웨어 단에서 디지털 변환된 값에 배수를 곱해 증폭**.
- **어퍼쳐 게인** = PA 장비에서 사용하는 항목, **현재 장비에서는 미사용**.
- **전체 게인** = 아날로그 게인 + 기타 값 합산, **데시벨(dB)** 단위 표현.
- 화면상 신호 크기는 **세로축 100% 기준 퍼센트(%)**로 환산 표시. 게이트 설정 시 신호 % 값과 깊이 정보가 함께 출력.

**샘플링 주파수 · 비트 해상도**
- 샘플링 주파수는 **파장 분해능**을 결정, **사용 주파수의 최소 5배** 이상(예: **5MHz 프로브 → 최소 25MHz 샘플링**). 하나의 파장을 정확히 구현하려면 **최소 5개 샘플링 포인트** 필요.
- **비트 해상도**: **16비트 권장**(신호 정보값 多). **8비트**는 신호값을 색상으로 치환해 비트맵으로 시각화. C-scan 저장 시 **16비트 데이터를 BMP/JPEG로 변환**해 파일 용량 절감.

**PRF와 검사 속도**
- **PRF(Pulse Repetition Frequency)** = 초당 펄스 발사 횟수. **검사 속도와 직결**.
- 예: **10,000Hz(초당 만 번) → 1mm당 1개 초음파 조건에서 초당 10m 이동(검사) 가능**.
- 철강 검사 현장은 **초당 10m 빠른 이동**이 요구되어 PRF를 높여야 함(주기를 짧게 해야 데이터가 끊기지 않고 연속 기록).
- **데이터 양과 트레이드오프**: 초당 샘플링이 높을수록 반응 빠르나 데이터량 증가. 검사 범위가 넓으면 신호 도달 시간으로 대기 발생 → 기구적 한계에 맞춰 대기 시간 최적화.

**딜레이 · 영점(원점) 설정**
- **딜레이**: **웨지 또는 갭에 의한 신호 지연 보정**. **원점(0점) 설정의 핵심**.
- **아크릴 웨지**에 의한 신호 지연을 보정하여 **첫 번째 신호를 영점 위치로 조정**. (손가락으로 뼈 신호 측정 시 발생하는 지연을 제거해 정확한 영점 기준을 잡는 것과 유사)
- **단일 게이트 사용 시 영점부터 게이트까지 거리를 측정하여 두께를 산출**. 교정 툴에 **영점 탐색 기능** 포함, X축 범위 mm 단위 설정.
- (현황: 딜레이·랭스 설정의 중요성 강조, 현재 딜레이 0 상태 확인. 탐촉자 주파수와 펄스 주파수를 동기화하는 공진 설정도.)

**음속 · 딜레이 자동 교정 (단일 게이트)**
- **음속 교정**: **두 에코 지점의 TOF(Time-of-Flight) 값으로 음속·딜레이 자동 계산**.
- 첫 번째 신호 캡처 → TOF 획득 → 게이트를 이동해 두 번째 신호 캡처 → **음속 자동 계산**. 두 게이트 값으로 **딜레이 산출 → 글로벌 설정 적용**.
- **다중 게이트가 아니라 단일 게이트**로 음속·딜레이 계산. 딜레이·음속 적용 시 **A-scan 값이 실시간 연동 변경**. 에코 원·에코 투의 거리 값을 지정하면 **원점 자동 계산·딜레이 조정**.
- 두 점의 음속·피크(peak) 값·A-scan 데이터로 음속·원점 계산. **A-scan 가로축 단위 = 밀리미터 / 포인트 개수 / 시간** 중 선택.
- **각 프로브마다 동일한 교정을 개별 수행**. **이론이 아닌 현장 작업자가 직접 수행하는 필수 과정**.

**TCG / DAC (거리별 감쇠 보정)**
- **DAC와 TCG** = 초음파가 **이동 거리에 따라 감쇄되는 현상을 보정**하여 증폭도 조절. 깊이별 신호 크기 차이를 보정 → **동일 크기 결함이 표면/바닥 어디에 있든 일관된 신호로 표현**. 미보정 시 C-scan에서 깊이별 신호 차이로 결함 크기 왜곡.
- **TCG(Time Corrected Gain)**: 깊이별 감쇠 보정 **슬로프 자동 생성**. **구매 사양서에 필수 기능으로 명시**.
- C-scan 표현을 위해 게이트 설정 + TCG 활성 → 특정 위치 신호 확인 후 게인을 **80% 수준으로 조정** → 게이트를 이동하며 각 지점 신호 포착·슬로프 조정으로 **신호 강도 일정 유지**. 반복 교정으로 슬로프 설정·파형 일치.

**필터 · 평균화(Average)**
- **밴드 패스 필터**: 특정 주파수 대역 외 신호 감쇄로 신호 정제(원하는 주파수 대역 선택·제어).
- **평균화(Average)**: 복수 사이클 평균화로 **전기 노이즈 제거**. 설정한 사이클 수만큼 평균 내어 파형 표현. **숫자(횟수) 증가 시 연산량↑ → 수집 속도(검사 속도) 저하**.

**펄스 전압**
- 사용 보드는 **최대 400V까지 출력 지원**. 전압을 높여 센서 감지 신호 강도 증폭. **펄스 에코 및 Through Transmission(투과) 모드**에서 교정 수행.

**게이트 · Threshold · 데이터 저장**
- **하드웨어에서 최대 4개 게이트 생성 가능하나 실제로는 하나만 활성화** 사용. 게이트 탭에서 마우스로 위치 조정·뒷부분 드래그로 범위 변경.
- **채널별 프로브 On/Off** 기능 필요. **Inspection Threshold**로 검사 게이트 높이 조정.
- **RF(라디오 프리퀀시) 신호를 정렬·합산하여 플러스 영역으로 표현하는 렉티피케이션(Rectification)** 과정.
- **데이터 저장**: 전체 포인트 저장 여부 / 게이트 구간 저장 여부 선택. TOF 데이터·A-scan 저장 필요성 논의. **C-scan 기본 저장 간격 1mm**. 엔코더와 무관하게 **PRF 설정에 따라 초당 1,000번 신호 지속 발생**. C-scan 표현 시 모든 데이터를 실시간 출력하는 대신 **엔코더 이동 거리에 맞춰 특정 지점만 샘플링** 표시. **타임 트리거 + 엔코더 값을 동시 업데이트하는 복합 트리거** 방식. 그래픽 스케일 **+100 ~ -100** 조정.
- **LUT(루티) 세팅을 모든 프로브에 일괄 적용**하는 기능.

**타임 슬롯 · 랭스(Range)**
- 검사 대상 두께 약 **300mm(30cm)** 환경에서 타임 슬롯 랭스 설정 범위 확인. 타임 슬롯 지정에 따라 **신호 가시 범위가 달라짐** → 검사 두께에 최적화된 설정 필요. (기존 장비는 조정 제한) → 신규는 **타임 슬롯을 사용자가 직접 지정**하도록 자유도↑.

**법규 기준 (80% / 14dB / 원자력)**
- **원자력 관련 법규**에 따라 **바닥 반사 신호를 80%에 맞추고 14dB를 올려 검사**하는 절차가 고정. (앰프 80% 설정 후 실제 검사 단계에서 14dB 증폭)
- **80%로 맞추는 이유** = 신호 포화(Saturation) 방지. 과거 장비는 100% 초과 시 포화·데이터 손실 → **20% 여유**로 신호 변화를 정확히 감지. 최신 장비는 **800%까지 인식 가능하나 원자력 법규가 변경되지 않아** 80% 기준 유지.
- 원자력 분야 검사 법규는 **25년 전과 동일** 유지.
- **결함 인식 원리**: 임계값(Threshold) 설정으로 신호 변화를 감지. **정상 신호가 유지되다 게이트 구간에서 급격히 감소하는 경우를 결함으로 인식**. **펄스에코법·점현반사법**으로 단조 공정 중 발생하는 **기공 결함** 검사. **수직 검사를 메인**으로, 각도 조절은 선택 옵션.

**오스테나이트 스테인리스강 신호 감쇄**
- **오스테나이트 스테인리스강**은 재료 특성상 **내부 결함이 없어도 신호가 불규칙하게 감쇄**. 기존 프로그램과 달리 **이 감쇄 구간을 정확히 탐지하여 시스템에 시각적으로 표현**하는 것이 개발 목표. 검사 화면 아웃풋으로 신호 급소실 구간 식별·분석.

**자동 교정 요구 (두산에너빌리티)**
- 교정은 현재 **수동**으로 이루어지나 두산에너빌리티 현장은 **자동 교정 기능**을 요구.
- **첫 번째와 두 번째 에코의 높이를 80%로 일괄 조정**하는 기능 필요. 두산에너빌리티 측은 **시료판을 활용해 세 개 이상의 점을 찍는 방식의 자동 교정** 요구. TCG·필터 설정으로 주파수 대역 선택·제어 포함.

**자동 마킹 · 기타**
- 기존에 작업자가 직접 대차에 올라가 수동 수행하던 **마킹·검사를 자동화**하기 위해 투자 진행. 실시간 신호 변화 감지 → 외부 출력 → 마커 연동으로 검사 결과 자동 표시. 초음파 수신·생성 설정은 **컨피그레이션 매니저**를 통해 수행.
- **PAUT 모드**에서는 하나의 프로브에서 다수의 H-scan 생성(채널 표현으로 이해).
- 음속 설정·시각화는 **Steel 재질 기준**으로 진행. **Inspection 탭의 Configuration 메뉴**에서 프로브 주파수·위치로 송수신 방식 정의 → Configuration 값을 프로브에 적용. Configuration은 각 프로브 개별 생성, 홀더 등록 프로브를 지정·매칭, 사용할 종파 지정으로 설정 완료.
- **2026년까지 임시 라이선스 발급 및 소프트웨어 사용 지원 예정**.

**[학생 질의]** 80%로 맞추는 이유는?
**[강사 답변]** 구형 장비의 **포화(Saturation) 방지를 위한 법규 기준**이며, 현재 장비는 800%까지 인식 가능하나 원자력 법규는 변경되지 않았다.

**[강사 강조]** “오늘 이해 못해요. 절대로 못하시니까 계속 반복하고 소통하면서 진행하세요.”

## 5. C-scan 표시 · 평가(Evaluation) · 리포트 및 기타 기능

검사 완료 후 C-scan 데이터를 기반으로 **결함 위치·면적·비율을 정량화**하고, **글로벌 dB 값을 사후 조정하여 전체 C-scan에 실시간 반영**하는 기능이 핵심 요건.

**글로벌 dB 사후 조정 (핵심 가치)**
- 법규상 **“전체 셸 중 가장 좋은 부분을 80%로 놓고 나머지를 측정”**해야 하나, **검사 전에 최적 지점을 알 수 없으므로 검사 후 dB 조정이 필수**.
- 현재 수동 방식은 수백 개 격자를 개별 조정 → **1건 검사에 1주일 소요**(비효율·위험) → **자동화가 이 프로젝트의 핵심 가치**.
- **저장된 로우 데이터 기반으로 dB 변경 시 전체 C-scan이 실시간 업데이트**(포토샵 밝기 조절 방식과 유사). 기존 데이터를 유지한 채 수치만 변경해 결과값 실시간 갱신. **분절된 데이터 단위를 통합**하고 dB 조절이 **글로벌하게 적용**되도록 구성.
- A-scan 교정 시 설정된 dB 값을 기준으로 **각 색상별 신호 강도를 글로벌 적용·수치화**해 표시. (피크 데이터의 로우 데이터 기준 적용 여부는 기술 스펙 검토 후 결정). **법적 코드 규정 준수를 위해 필수**.

**필수 뷰 · 화면 구성**
- **필수 뷰**: A-scan · B-scan · C-scan · **D-scan(= S-scan과 동일 개념)** · FFT · **Strip Chart**.
- **Strip Chart**: 검사 전 구간을 **한 줄(타임라인)로 펼쳐 표시**하여 결함 유무 일괄 확인. 정상 구간은 정상으로, 결함 구간은 별도 표시(직관적). (C-scan 외 전체 결과 요약 보기 요구 반영)
- **검사창 최대 5개** 구성·탭 추가로 다양한 레이아웃 생성. 각 뷰에 A/B/C-scan 등 표현 방식 지정 + 특정 프로브 신호 할당·배치. **교정·검사·평가 화면 탭 자유 추가**(과거 장비와 달리 메모리 제한 없이 확장). SDK 기능으로 목적에 맞는 화면 구성 후 저장. 검사 화면 가독성 위해 불필요 UI 제거·화면 확장 고려.

**평가(Evaluation) 화면**
- 결함 영역을 **커서로 지정**, **면적·위치 값 표시**, **전체 대비 결함 비율 산출**(스캔 영역 내 특정 부위 넓이·위치 정밀 표시, 수동 커서 측정).
- **데이터 저장·조회** 기능. **업체별 상이한 평가 기준을 커스텀 설정**으로 반영.
- **두산 요구**: C-scan 화면 포함, **한 바퀴 회전한 전체 데이터를 하나의 파일로 펼쳐 시각화**. **감도 저하 구간 등 특이사항을 화면에 명확 표시**.

**검사 시작/자동화 프로세스**
- **고정형·이동형 통합 지원** — 프로브·제품 이동 여부와 무관하게 두 환경 모두 적용(기존엔 별도였으나 상용은 통합).
- **검사 대기 상태에서 외부 트리거 신호 수신 시 자동으로 스캔 데이터 생성**. 시작 버튼은 준비 상태 활성, 실제 개시는 별도 트리거. 검사 중지 시 **데이터 저장 여부 확인** 절차.
- 철강 생산 라인에서 **PLC/PC 서버로 바코드 인식 → 검사 일련번호 자동 매칭**. 검사 완료 후 **합격/불합격 자동 판정 창**.

**리포트**
- 벤치·프로브·Configuration·A-scan 파라미터 등 **검사 이력 전체 포함**. **커스텀 리포트 / 엑셀 리포트** 지원(검사 데이터 자동 반영).
- NDT의 핵심 목적 = **검사 증거 확보로 제품 부가가치 제고**. **법적 증거력**을 위해 A-scan 데이터·설정값 등을 포함한 리포트 필수. 화면 캡처·세부 세팅 포함, 기존 프로그램 이상 성능. **직관·미려한 레이아웃**, 상용화 가능 수준(중공업 등 현장 요구 반영).

**계정 권한 · AI 매뉴얼**
- **계정 권한 관리**: **초음파 전문가**(세팅 변경 가능) / **현장 작업자**(세팅 변경·데이터 삭제 불가) 단계별 접근 권한으로 데이터 보호.
- **AI 대화형 매뉴얼**: 신규 SW에 탑재. **한글로 기능 설명 제공**(물음표 아이콘으로 대화형 질의). **인터넷 연결 필요**(시스템 리셋 시 연결 상태 재확인).

**확장성**
- 두산은 기존에 사용하는 **모든 장비와 소프트웨어를 통합할 계획** → 개발 SW는 **범용성**을 갖춰야 하며, 초기 개발 단계부터 향후 확장성을 고려해 기능 목록 선정.

**[강사 강조]** 글로벌 dB 조정 기능은 **항공우주 분야로도 확장 가능한 핵심 기능**이므로 반드시 구현할 것 — “무조건 된다고 얘기를 해놨다.”

## 부록 — 이 강의의 Key Points

- 기존 상용 SW의 전체 기능 흐름(벤치 → 프로브 홀더 → Input/Output → A-scan 교정 → C-scan 평가 → 리포트)을 이해하고, 신규 ERUT 스캔형은 이 **기능 수준 이상**으로 개발.
- **로봇 좌표 기반 엔코더 신호 변환(6축 → 3축 → 펄스)** 구조는 기존 물리 엔코더 방식과 다르므로, **엔코더 캘리브레이션 자동화** 기능을 별도 구현.
- **TCG 자동 교정 및 글로벌 dB 사후 조정**은 법규 준수와 검사 자동화의 핵심이며 **두산 요구사항으로 반드시 구현**.
- **확장성 설계 중요** — 두산은 셸 검사 외에도 보유한 모든 장비·SW를 이 시스템으로 통합 계획이므로 **범용 구조**로 개발.
